# ShopFloor Agent: Claude AI Integration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Wire Claude into the ShopFloor backend so the agent generates intelligent content (quotes, emails, CRM notes, completion summaries) and delegates execution to Scalekit as the appropriate persona, proving delegated-auth with real LLM-generated workflows.

**Architecture:** Create a new `ai_service.py` module that wraps Claude calls and exposes typed prompt/completion functions. Each endpoint calls the AI service to draft content, then passes that content to Scalekit execution functions (which don't stub anymore—they actually call tools). Identity stays resolved by the backend from the session cookie; the LLM never sees or chooses it.

**Tech Stack:** Claude 3.5 Sonnet via `anthropic` SDK, structured prompts, existing Scalekit wrapper, audit logging.

import os
from typing import Optional

from anthropic import Anthropic

_client: Optional[Anthropic] = None


def get_claude_client() -> Anthropic:
    global _client
    if _client is None:
        api_key = os.environ.get("ANTHROPIC_API_KEY")
        if not api_key:
            raise RuntimeError("ANTHROPIC_API_KEY not set")
        _client = Anthropic(api_key=api_key)
    return _client


def get_model() -> str:
    return os.environ.get("CLAUDE_MODEL", "claude-3-5-sonnet-20241022")


def get_temperature() -> float:
    return float(os.environ.get("CLAUDE_TEMPERATURE", "0.7"))


def get_max_tokens() -> int:
    return int(os.environ.get("CLAUDE_MAX_TOKENS", "1024"))


def completion(system: str, user_message: str) -> str:
    """Call Claude and return text response."""
    client = get_claude_client()
    response = client.messages.create(
        model=get_model(),
        max_tokens=get_max_tokens(),
        temperature=get_temperature(),
        system=system,
        messages=[{"role": "user", "content": user_message}],
    )
    return response.content[0].text
```

def draft_quote(vehicle: str, symptom: str, concern: str) -> QuoteDraft:
    query = f"{vehicle} {symptom} {concern}".strip()

    risk_keywords = {"noise", "leak", "warning light", "intermittent", "grinding", "squealing"}
    risk_multiplier = 1.15 if any(k in query.lower() for k in risk_keywords) else 1.0

    if _is_live():
        try:
            comparables = tuple(_query_actian(query, limit=3))
            detail = "Actian VectorAI retrieval: top 3 semantic matches from repair history."
        except Exception as exc:
            comparables = _stub_comparables()
            detail = f"Actian retrieval failed ({exc}); showing cached comparables."
    else:
        comparables = _stub_comparables()
        detail = "Stub mode: showing seeded comparables (set ACTIAN_MODE=real on Render)."

    quote_amount = round(mean(job.final_price for job in comparables) * risk_multiplier)
    
    # Use Claude to generate quote text
    try:
        from app.services.claude import generate_quote_text
        comparable_prices = [job.final_price for job in comparables]
        ai_quote = generate_quote_text(vehicle, symptom, comparable_prices, quote_amount)
        quote_text = ai_quote.quote_text
    except Exception as exc:
        # Fallback to template if Claude fails
        quote_text = (
            f"Based on {len(comparables)} similar repairs for {vehicle}, "
            f"we estimate ${quote_amount} to address: {symptom or concern}."
        )
        detail += f" (Claude generation failed: {exc})"

    return QuoteDraft(
        quote_amount=quote_amount,
        quote_text=quote_text,
        comparables=comparables,
        detail=detail,
    )

# ShopFloor AI Integration

## Overview

The ShopFloor agent uses Claude 3.5 Sonnet to generate intelligent, contextual content:

- **Quote Text**: Drafted from comparable historical repair jobs
- **Customer Emails**: Professional, personalized repair communications
- **CRM Approval Notes**: Manager-written approval and status notes
- **Completion Summaries**: Technician-written work summaries

## Architecture

### Identity Delegation Model

1. **Backend resolves identity** from session cookie (`demo_actor_id`)
2. **Claude generates content** (never sees or decides identity)
3. **Backend executes tool** through Scalekit **as the resolved actor**
4. **Audit records** the specific person, tool, and outcome

This ensures: "The model never gets a vote on whose token it uses."

### Content Generation Functions

#### `generate_quote_text(vehicle, symptom, comparable_prices, final_quote_amount)`
Returns: `QuoteContent` with `quote_text` and `reasoning`

#### `generate_customer_email(customer_name, vehicle, quote_amount, quote_text)`
Returns: `EmailContent` with `subject` and `body`

#### `generate_approval_notes(vehicle, symptom, quote_amount, customer_name)`
Returns: `CRMNotes` with `title` and `body`

#### `generate_completion_summary(vehicle, symptom, work_description)`
Returns: `CompletionNotes` with `summary` and `next_steps`

## Configuration

Add to `.env`:

```
ANTHROPIC_API_KEY=sk-ant-...
CLAUDE_MODEL=claude-3-5-sonnet-20241022
CLAUDE_TEMPERATURE=0.7
CLAUDE_MAX_TOKENS=1024
```

## Fallback Behavior

If Claude fails (network error, rate limit, bad key), the agent:
1. Logs the error to detail field
2. Falls back to deterministic template text
3. Continues execution (degrades gracefully)
4. Audit records the fallback in detail

## Testing

- **Mock tests** use `monkeypatch` to stub Claude responses
- **Real tests** require `ANTHROPIC_API_KEY` and are skipped in CI if key unavailable
- All tests verify content is reasonable (not empty, contains expected keywords)

## Deployment

On Render:
1. Set `ANTHROPIC_API_KEY` in environment variables
2. Claude calls happen at quote/approval/completion time (no startup delay)
3. Monitor Claude errors in audit detail fields
4. If consistently failing, check quota and API key validity
```
